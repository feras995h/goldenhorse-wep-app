# تحسينات النظام المحاسبي

## نظرة عامة
تم تطوير وتحسين النظام المحاسبي ليشمل جميع المتطلبات المذكورة مع واجهة مستخدم محسنة وتجربة مستخدم أفضل.

## التحسينات المضافة

### 1. دليل الحسابات (شجرة الحسابات)
**المسار:** `/financial/chart-of-accounts`

**الميزات:**
- عرض الحسابات كشجرة هرمية
- البحث في الحسابات بالرقم أو الاسم
- تصفية حسب نوع الحساب
- إضافة وتعديل الحسابات
- تصدير دليل الحسابات
- عرض رصيد كل حساب

### 2. القيود اليومية المحسنة
**المسار:** `/financial/journal`

**الميزات الجديدة:**
- **البيان خانة إجبارية** - لا يمكن حفظ القيد بدون بيان
- **التاريخ تلقائي** - يتم تعيين تاريخ اليوم تلقائياً مع إمكانية التعديل
- **البحث التلقائي في الحسابات** - عند كتابة رقم الحساب يظهر الاسم تلقائياً والعكس
- **التوازن التلقائي** - التحقق من تساوي المدين والدائن
- **منع المدين والدائن في نفس السطر** - عند إدخال مدين يتم مسح الدائن والعكس
- **عرض رصيد الحساب** - يظهر رصيد الحساب بعد إجراء القيد
- **حساب الإجماليات** - إجمالي المدين والدائن مع سعر الصرف
- **المعدل (سعر الصرف)** - تلقائي 1 مع إمكانية التعديل
- **الإضافة التلقائية للأسطر** - إضافة سطر جديد عند ملء آخر سطر

### 3. كشف الحساب المحسن
**المسار:** `/financial/account-statement`

**الميزات:**
- البحث في الحسابات بالرقم أو الاسم
- تحديد فترة زمنية (من تاريخ - إلى تاريخ)
- خيار "كل الفترات" لحساب من بداية المدة
- عرض الرصيد الافتتاحي والختامي
- تصدير كشف الحساب
- طباعة الكشف

### 4. الأصول الثابتة المحسنة
**المسار:** `/financial/fixed-assets`

**الميزات الجديدة:**
- **رقم الأصل والأصل والفئة** - معلومات أساسية للأصل
- **التكلفة ومجمع الإهلاك والقيمة الدفترية** - حساب تلقائي
- **الحالة** - نشط، متخلص، خردة
- **الخردة** - قيمة الخردة مع الحساب التلقائي
- **المعادلة:** التكلفة - الخردة = القيمة الدفترية + مجمع الإهلاك
- **حساب الإهلاك التلقائي** - حسب طريقة الإهلاك المختارة

### 5. القيد الافتتاحي (الأرصدة الافتتاحية)
**المسار:** `/financial/opening-balance`

**الميزات:**
- إضافة أرصدة افتتاحية للحسابات
- التحقق من توازن الأرصدة
- استيراد وتصدير الأرصدة الافتتاحية
- إنشاء قيد افتتاحي تلقائي
- مراقبة الأرصدة الافتتاحية

### 6. مراقبة الحسابات
**المسار:** `/financial/account-monitoring`

**الميزات:**
- إضافة حسابات للمراقبة
- تحديد تكرار المراقبة (يومي، أسبوعي، شهري)
- تحديد حد أدنى للتغيير
- تنبيهات عند الزيادة أو الانخفاض
- عرض التغييرات في الأرصدة
- إحصائيات المراقبة

### 7. تقارير الفواتير
**المسار:** `/financial/invoice-reports`

**الميزات:**
- **فواتير مسددة بالكامل** - عرض وإحصائيات
- **فواتير مسددة جزئياً** - عرض وإحصائيات  
- **فواتير غير مسددة** - عرض وإحصائيات
- **فواتير متأخرة** - عرض وإحصائيات
- تصفية حسب التاريخ والنوع والحالة
- تصدير التقارير
- إحصائيات شاملة

## المكونات الجديدة

### 1. JournalEntryForm
مكون محسن لنموذج القيود اليومية مع جميع المتطلبات المذكورة.

### 2. FixedAssetForm  
مكون محسن لنموذج الأصول الثابتة مع الحسابات التلقائية.

### 3. ChartOfAccounts
صفحة جديدة لعرض دليل الحسابات كشجرة.

### 4. OpeningBalanceEntry
صفحة جديدة لإدارة الأرصدة الافتتاحية.

### 5. AccountMonitoring
صفحة جديدة لمراقبة الحسابات.

### 6. InvoiceReports
صفحة جديدة لتقارير الفواتير.

## التحسينات التقنية

### 1. واجهة المستخدم
- تصميم متجاوب ومحسن
- ألوان متناسقة مع النظام
- أيقونات واضحة ومعبرة
- تجربة مستخدم سلسة

### 2. الأداء
- تحميل البيانات بشكل تدريجي
- البحث الفوري في الحسابات
- حساب الإجماليات في الوقت الفعلي
- تحسين استعلامات قاعدة البيانات

### 3. الأمان
- التحقق من صحة البيانات
- منع إدخال قيم غير صحيحة
- التحقق من التوازن في القيود
- حماية من الأخطاء البرمجية

## كيفية الاستخدام

### 1. إنشاء قيد يومية جديد
1. انتقل إلى "قيود اليومية"
2. اضغط على "إضافة قيد جديد"
3. أدخل البيان (إجباري)
4. اختر الحسابات بالبحث التلقائي
5. أدخل المبالغ (مدين أو دائن)
6. تأكد من التوازن
7. احفظ القيد

### 2. عرض دليل الحسابات
1. انتقل إلى "دليل الحسابات"
2. استخدم البحث للعثور على الحسابات
3. اضغط على الحساب لعرض التفاصيل
4. استخدم التصفية حسب النوع

### 3. إنشاء كشف حساب
1. انتقل إلى "كشف الحساب"
2. اختر الحساب بالبحث
3. حدد الفترة الزمنية
4. اضغط على "عرض الكشف"

### 4. إدارة الأصول الثابتة
1. انتقل إلى "الأصول الثابتة"
2. اضغط على "إضافة أصل جديد"
3. أدخل المعلومات الأساسية
4. أدخل التكلفة والخردة
5. احفظ الأصل

### 5. إنشاء قيد افتتاحي
1. انتقل إلى "القيد الافتتاحي"
2. أضف الأرصدة الافتتاحية
3. تأكد من التوازن
4. اضغط على "إنشاء القيد الافتتاحي"

## الملاحظات المهمة

1. **التوازن إجباري** - لا يمكن حفظ قيد غير متوازن
2. **البيان إجباري** - لا يمكن حفظ قيد بدون بيان
3. **التاريخ تلقائي** - يتم تعيين تاريخ اليوم تلقائياً
4. **البحث التلقائي** - اكتب رقم الحساب أو الاسم للبحث
5. **الحسابات التلقائية** - يتم حساب الإهلاك والقيم تلقائياً

## الدعم الفني

في حالة وجود أي مشاكل أو استفسارات، يرجى التواصل مع فريق الدعم الفني.
