# الواجهة المبسطة لإضافة الحسابات

## نظرة عامة

تم تبسيط واجهة إضافة الحسابات بشكل كبير لتكون أكثر سهولة في الاستخدام وأقل تعقيداً، مع الاحتفاظ بجميع الوظائف الأساسية المطلوبة.

## التغييرات المطبقة

### 1. إزالة التعقيدات غير الضرورية
- **قبل التبسيط**: 6 أقسام منفصلة مع ألوان مختلفة
- **بعد التبسيط**: 3 أقسام بسيطة ومباشرة

### 2. تبسيط المعلومات الأساسية
- **الحقول المطلوبة فقط**:
  - رقم الحساب (إجباري)
  - اسم الحساب (إجباري)
  - التصنيف (إجباري)
  - عملة الحساب (اختياري)

- **الحقول المحذوفة**:
  - الاسم بالإنجليزية (غير ضروري)
  - نوع الحساب (يتم تحديده تلقائياً)
  - المستوى (يتم تحديده تلقائياً)
  - طبيعة الحساب (يتم تحديدها تلقائياً)
  - حالة الحساب (افتراضية: نشط)

### 3. تبسيط اختيار الحساب الأب
- **قبل التبسيط**: قسم معقد مع أزرار وألوان متعددة
- **بعد التبسيط**: حقل بحث بسيط مع قائمة منسدلة

### 4. تبسيط الوصف
- **قبل التبسيط**: قسمين منفصلين للوصف والملاحظات
- **بعد التبسيط**: حقل وصف واحد بسيط

## الواجهة الجديدة المبسطة

### 1. المعلومات الأساسية
```
┌─────────────────────────────────────────────────────────┐
│ رقم الحساب *        │ اسم الحساب *                      │
│ [مثال: 1001]        │ [مثال: النقد في البنك]            │
├─────────────────────────────────────────────────────────┤
│ التصنيف *           │ عملة الحساب                        │
│ [اختر التصنيف]      │ [دينار ليبي (LYD)]                │

```

### 2. الحساب الأب (اختياري)
```
┌─────────────────────────────────────────────────────────┐

│ 2000 - المصروفات (مصروفات)                             │

```

```

```

## الميزات المحتفظ بها

### 1. الترقيم التلقائي
- **الحسابات الرئيسية**: ترقيم من 1000، 2000، 3000، إلخ
- **الحسابات الفرعية**: ترقيم هرمي (الأب.001، الأب.002، إلخ)

### 2. اختيار الحساب الأب
- قائمة منسدلة بسيطة
- عرض جميع الحسابات الرئيسية
- عرض الكود والاسم والتصنيف لكل حساب

### 3. التحديد التلقائي للخصائص
- **طبيعة الحساب**: مدين/دائن حسب التصنيف
- **نوع الحساب**: رئيسي/فرعي حسب وجود الحساب الأب
- **المستوى**: يتم تحديده تلقائياً

### 4. التحقق من صحة البيانات
- التحقق من الحقول المطلوبة
- منع تكرار أرقام الحسابات
- التحقق من صحة الحساب الأب

## الفوائد من التبسيط

### 1. سهولة الاستخدام
- **وقت أقل**: إدخال أسرع للحسابات
- **أخطاء أقل**: عدد أقل من الحقول المطلوبة
- **وضوح أكبر**: تركيز على المعلومات الأساسية

### 2. تجربة مستخدم محسنة
- **واجهة نظيفة**: تصميم بسيط ومباشر
- **تنقل سهل**: أقل أقسام للتنقل بينها
- **فهم أسرع**: معلومات واضحة ومباشرة

### 3. كفاءة أعلى
- **إنتاجية أكبر**: إنشاء حسابات أسرع
- **تدريب أسهل**: تعلم أسرع للمستخدمين الجدد
- **صيانة أقل**: كود أبسط وأسهل في الصيانة

## كيفية الاستخدام

### 1. إنشاء حساب رئيسي
```
1. املأ رقم الحساب (مثال: 1001)
2. املأ اسم الحساب (مثال: النقد في البنك)
3. اختر التصنيف (مثال: أصول)
4. اختر العملة (اختياري)
5. اترك الحساب الأب فارغاً
6. أضف وصف (اختياري)
7. احفظ الحساب
```

### 2. إنشاء حساب فرعي
```
1. املأ اسم الحساب
2. اختر التصنيف
3. اختر العملة (اختياري)
4. اختر الحساب الأب من القائمة المنسدلة
5. أضف وصف (اختياري)
6. احفظ الحساب
```

## أمثلة على الاستخدام

### مثال 1: إنشاء حساب أصول
```
رقم الحساب: 1100
اسم الحساب: الأصول المتداولة
التصنيف: أصول
العملة: دينار ليبي (LYD)
الحساب الأب: (فارغ - حساب رئيسي)
الوصف: الأصول التي يمكن تحويلها إلى نقد خلال سنة
```

### مثال 2: إنشاء حساب فرعي
```
اسم الحساب: النقد في البنك
التصنيف: أصول
العملة: دينار ليبي (LYD)
الحساب الأب: 1000 - الأصول (من القائمة المنسدلة)
الوصف: النقدية المودعة في البنك
```

## التحسينات المستقبلية

1. **حفظ القوالب**: حفظ قوالب للحسابات المتكررة
2. **استيراد جماعي**: استيراد عدة حسابات من ملف
3. **نسخ الحسابات**: نسخ حساب موجود مع تعديل الرقم
4. **اقتراحات ذكية**: اقتراح أرقام وأسماء الحسابات
5. **تحقق متقدم**: تحقق من صحة البيانات أثناء الكتابة

## الخلاصة

الواجهة المبسطة تجعل إضافة الحسابات أسرع وأسهل وأقل عرضة للأخطاء، مع الاحتفاظ بجميع الوظائف الأساسية المطلوبة. التصميم الجديد يركز على سهولة الاستخدام والكفاءة.
